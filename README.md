# M001 シミュレータ

物理配線モデル（LED、モーター、各種センサー）を画面上およびOPC UAサーバーを介して再現・検証するためのシミュレーションシステムです。

##  システム概要

本システムは以下の3つのコンポーネントで構成されています。

1. **メインロジック & OPC UAサーバー (`main.py`)**
   - GPIO（モック環境）を制御するバックエンドロジック。
   - センサー状態の変化を検知し、アクチュエータ（モーターやLED）のインターロックを制御します。
   - 状態をOPC UAサーバー経由で外部へ公開・同期します。

2. **Streamlit UI シミュレータ (`app.py`)**
   - ブラウザ上で直感的にセンサーのHIGH/LOWを切り替え、アクチュエータの稼働状態をビジュアルで確認できるWebアプリケーションです。
   - 事前に用意されたシナリオ（テストケース）をワンクリックで再現するパネルも備えています。

3. **自動テスト (`test_main.py`)**
   - `pytest` を使用し、制御ロジックの単体テスト、OPC UAのノード階層およびデータ同期の結合テスト、メインループの正常終了テストを自動化しています。

---

## 🛠 動作環境・依存ライブラリ

- **Python**: 3.9 / 3.10 / 3.11
- **主要ライブラリ**:
  - `streamlit` (Web UI用)
  - `gpiozero` (GPIOデバイス制御・モック用)
  - `opcua` (OPC UA サーバー/クライアント通信用)
  - `pytest` (テスト実行用)

---

## 🚀 使い方

### 1. 依存ライブラリのインストール
```bash
pip install streamlit gpiozero opcua pytest

```

### 2. Streamlit Webシミュレータの起動

画面上でセンサーの状態をトグル切り替えし、挙動をシミュレートします。

```bash
streamlit run app.py

```

起動後、ブラウザで `http://localhost:8501` にアクセスしてください。

### 3. OPC UA サーバー（メインループ）の起動

スタンドアロンでOPC UA サーバーを起動し、1秒周期で同期ループを実行します。

```bash
python main.py

```

* サーバーエンドポイント: `opc.tcp://0.0.0.0:4840/freeopcua/server/`

### 4. テストの実行

`pytest` を使用して、すべてのテストシナリオを検証します。

```bash
pytest test_main.py -v

```

## 📌 ピン配置と仕様（プルアップ回路）

| デバイス名 | 種別 | ピン番号 | 通常時 (ON) | 変化時 (OFF) |
| --- | --- | --- | --- | --- |
| **フォトトランジスタ** | 入力 | Pin 5 | HIGH (1): 受光中 | LOW (0): 光遮断 |
| **磁気センサー** | 入力 | Pin 6 | HIGH (1): 磁石なし | LOW (0): 磁石検出 |
| **プッシュスイッチ** | 入力 | Pin 16 | HIGH (1): 押してない | LOW (0): 押下 |
| **コンベアモーター** | 出力 | Pin 22 | HIGH (1): 回転中 | LOW (0): 停止 |
| **投光器 LED01** | 出力 | Pin 23 | HIGH (1): 点灯中 | LOW (0): 消灯 |
| **ステータス LED02** | 出力 | Pin 25 | HIGH (1): 点灯中 | LOW (0): 消灯 |
